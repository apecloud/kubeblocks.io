测试说明

数据库版本升级测试包括数据库大版本升级和小版本升级，由于数据库大版本升级可能涉及到数据结构变化等带来的不兼容的问题，每个引擎的情况大不相同，生产环境一般建议首先进行兼容性测试，再酌情采用升级或者迁移的方式，因此，大版本升级测试不在本测试范围内。

本测试涵盖主要引擎的小版本升级，利用Kubeblocks提供的能力，通过修改cluster yaml的serviceversion来实现小版本的升级、降级等操作，并验证操作后数据库状态正常，数据完整。

测试方法

通过直接修改cluster yaml中的serviceVersion的值，或者提交如下修改serviceVersion的OpsRequest，来触发升级mysql的小版本到8.0.39

```yaml
apiVersion: operations.kubeblocks.io/v1alpha1
kind: OpsRequest
metadata:
  name: mysql-example-upgrade-cmpv
  namespace: examplens
spec:
  clusterName: mysql-example
  upgrade:
    components:
    - componentName: mysql
      serviceVersion: 8.0.39
  type: Upgrade
```

测试结果

升级/降级后，数据库状态正常，数据完整，测试通过。
